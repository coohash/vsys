VSYS Volume Booster & Liquidity Simulation Engine

## 🧭 项目简介

**VSYS Volume Booster** 是一款专门针对 VSYS 公链（v.systems）生态量身定制的高性能、分布式链上活跃度与流动性深度模拟引擎。

本引擎彻底告别了传统刷量脚本“固定金额、全额扫尾、固定间隔”的机械化特征，独创了**多层级资金拓扑网络**与**长尾随机残留对抗算法**。通过高精度二进制本底交易组装和全异步非阻塞轮询，在公链账本上完美模拟数万级真实用户的日常交互行为，全面激活生态链上数据，提升公链的交易笔数（Tx Count）与活跃持币地址数。

## 💡 解决的核心痛点

在公链生态运营及数据维护中，传统的链上数据补全脚本通常面临以下致命痛点，本系统对此进行了全方位攻克：

1. **链上行为高度可预测**
   * *传统痛点*：转账金额多为整数，或小数点位数固定；钱包归集时常常“全额扫尾”将余额直接清零。在链上图谱中会呈现极具特征的“扇入-扇出”星型拓扑，极易被各大区块链浏览器和数据分析机构判断。
   * *本项目解法*：引入**动态多精度漂移技术**（分发金额 2~8 位小数随机变换）与**长尾资金随机残留机制**（归集时强制保留 “设定数量” VSYS 且保留额的小数位随机），使钱包余额状态长期保持真实波动，完美隐藏 交易 特征。
2. **长效高频运行下的资金链断裂与死锁**
   * *传统痛点*：多账户间并发错综复杂，极易因为某些节点资金耗尽导致整个自动化链路卡死，需要频繁人工干预补给。
   * *本项目解法*：内置 **L0级水源自动补给与自愈机制**。当核心中转节点水位不足时，自动触发向上层总金库（L0 级）的差额智能整型补给，形成无感知的防断裂资金流闭环。
3. **传统浮点数计算导致的财务账目“微小溢出”**
   * *传统痛点*：JavaScript/Python 的标准 `float` 在高频乘以 $10^8$ 转换为聪（Satoshi）时，会产生诸如 `0.00000001` 的精度丢失，导致高频链上广播因“金额不合规”或“余额不足”频频报错崩溃。
   * *本项目解法*：全面上锁 **Decimal 高精算子**，强制锁定 20 位有效数字精度，全面阻断金融级链上模拟产生的任何微小浮点数溢出，确保 Satoshi 级（$10^{-8}$）零误差。
4. **巨量高频并发下节点的通信过载与局部代理干扰**
   * *传统痛点*：传统的单线程同步网络请求对节点压力极大，且易受宿主机局部“代理/科学上网环境”的干扰导致 SSL 握手或重定向报错。
   * *本项目解法*：基于 `asyncio` 构建纯异步高性能状态机模型，持久化复用 `requests.Session` 连接池，同时显式屏蔽环境局部代理，保障 7×24 小时高并发、低延迟的稳定通信。

## 🏗️ 架构原理与资金拓扑流

本引擎采用 **L0总金库 ➔ 中转大户站 ➔ 散户独立地址** 的三层递进式资金拓扑流。通过概率学随机决策环（可设置 概率拆分分发，可设置 概率长尾归集），在公链上构建复杂博弈网络。

                +---------------------------------------+
                |        L0 Total Treasury (总金库)      |
                +-------------------+-------------------+
                                    |
                         (自愈补给: REFILL_MIN)
                                    v
                +-------------------+-------------------+
                |      Transit Accounts (中转大户站)     |
                +-------------------+-------------------+
                                    |
             (随机精度分发: TX_MIN ~ TX_MAX | 2-8位小数)
                                    v
                +-------------------+-------------------+
                |    Retail/Independent Addresses (散户) |
                +-------------------+-------------------+
                                    |
        (长尾随机残留归集: 保留 “你设定数量” VSYS 后回收至中转站)
                                    +-------------------+

整个生命周期分为以下四个闭环阶段：

1. **分发阶段（Distribution）**：中转大户站随机抽取独立地址，执行 2-8 位随机精度、大小在 `TX_MIN` 到 `TX_MAX` 之间的分散式资金划拨，瞬间激活下游散户地址的链上流水。
2. **自愈阶段（Self-Healing）**：当中转站地址的可用余额干涸（低于 `TX_MIN + 0.1 VSYS`）时，系统自动向上触发 L0 总金库的差额整型补给（金额随机介于 `REFILL_MIN` 到 `REFILL_CAP` 之间），直到金库耗尽。
3. **归集阶段（Collection）**：当独立散户地址的资产超过 `COLLECT_THRESHOLD` 警戒线时，自动触发随机长尾残留回收。系统绝不全额扫尾，而是随机留下 “你设定数量”  VSYS，其余资金安全回收至中转大户，供下一轮流动性循环使用。
4. **永续阶段（Perpetual Loop）**：若总金库（L0）弹药完全枯竭，系统会自动无缝切换为中转站与独立地址群之间的内循环闭环模式——“分发-归集-再分发”，确保脚本在零外接水源的情况下依然能永久不间断运转。

## 🛠️ 实现方式与核心技术亮点

1. **原生 Type 2 交易大端序拼接**
   项目直接调用底层 `struct.pack` 按照 VSYS 官方规范的大端序（`>`）将交易字节流（Type 2 基础交易：包含类型、19 位纳秒级全网时间戳、Satoshi 级转账量、矿工费率比等）进行物理压缩。不依赖任何重型 SDK，执行效率达到极致。
2. **自适应离线密码学签名**
   脚本内置对 `py_vsys` 顶层空间与底层路径的动态向下双层兼容探测。采用自适应签名绑定，自动提取 Curve25519 的签名句柄，在本地离线计算出密文签名流后再向外推送，保证私钥在整个签名生命周期中绝不联网。
3. **异步无限状态机控制环**
   主程序运行于 `asyncio` 的高层级异步死循环中。在成功向公链广播一笔交易后，通过可配的时间步调进行无阻塞休眠；若判断当前随机账户由于余额不足不满足转账条件，则通过 0.8（可设置） 秒极速微休眠瞬间切换状态，极大地节约了 CPU 算力损耗。
4. **可多个终端同时运行**
   进阶应用，可以制造地址交集来模拟真实世界错综复杂的转账网。进阶应用中，您可以同时开启 4-8 个终端实例协同运行。在运行一段时间后，通过更新 CSV 配置文件，使不同终端使用的账户库产生 25% - 33% 的重叠。这一策略可以打破单线资金流隔离，有效编织出高度逼真的链上蜘蛛网式资金交互图谱。

## ⚙️ 核心配置参数详解（请依据实际需求设置）

通过调节全局静态类 `Config` 中的各阈值参数，可以完美调优不同资金规模和冲量速率要求的公链生态网络：

| 参数名称 | 默认配置值 | 参数物理意义与调优建议 |
| :--- | :--- | :--- |
| `NODE_URL` | `"http://wallet-node.v.systems:9922"` | 目标链上数据通信 RPC 节点。大规模冲量时建议切换为自建高速 RPC 超级节点以应对百万级并发。 |
| `L0_ADDRESS` | `"ARRfwY...2WdMR"` | **L0级顶级总金库地址**：整个生态圈的底层水源。 |
| `L0_PRIV` | `"3kFxJq...FwCyn6Z"` | **L0级顶级总金库私钥**（生产环境请务必更换为你自己的私钥）。 |
| `TX_MIN` | `600.0` | 中转站向散户分发资金时的**单笔转账下限**（单位：VSYS）。建议与 MAX 保持一定区间。 |
| `TX_MAX` | `3600.0` | 中转站向散户分发资金时的**单笔转账上限**（单位：VSYS）。区间拉大、非整数组合可提升链上真实性。 |
| `REFILL_MIN` | `3600` | L0 金库向中转站发起自动整型补给的**单次补给下限**（必须是整数型，单位：VSYS）。 |
| `REFILL_CAP` | `6000` | L0 金库向中转站发起自动整型补给的**单次补给上限**（必须是整数型，单位：VSYS）。保证中转站水位充沛。 |
| `COLLECT_THRESHOLD`| `600.0` | 散户地址触发向中转站**归集回收的起征线**。**⚠️ 必须严格大于“随机残留上限(500) + 矿工费(0.1)”**。 |
| `FEE_SAT` | `10_000_000` | 协议级摩擦成本。VSYS 公链标准转账固定矿工费（10,000,000 Satoshi = 0.1 VSYS）。 |

### ⏱️ 速率与频次微调建议

脚本底部的异步休眠直接决定了全网流水的刷新频率。请在 `run` 函数中定位并调整以下逻辑：
* **模拟自然低频真实持币（默认）**：
  ```python
  if action_taken:
      await asyncio.sleep(random.randint(2, 13)) # 2~13秒随机休眠，日均约1万次转账，消耗约1000 VSYS手续费
  ```
* **极限高频冲量模式（需节点响应极快）**：：
  ```python
  if action_taken:
    await asyncio.sleep(random.uniform(0.1, 1.0)) # 0.1~1秒极速高频广播，适合节点性能强悍且急需冲高成交量的场景
  ```

🚀 快速上手与部署指引
# 1. 规模配比与数据准备
当总金库初始配置约 1,000,000 VSYS 资产时，生产级规模建议配比：

中转站地址（translate_midd.csv）：建议数量 < 500 个
独立散户地址（add_priv.csv）：建议数量 < 10,000 个
数据格式规范（极其重要）：
请使用批量工具（如参照 10000_VSYS_Address.txt 中的方法）生成好地址与私钥。
用记事本或文本编辑器分别创建 translate_midd.csv 和 add_priv.csv 文件，放入对应账户。
格式要求：一行一个，地址与私钥中间用英文逗号 , 隔开，绝对不能有标题行（Header）。

# 2. 前台临时测试运行（本地 PC）
本引擎需 7×24 小时长时间不间断轮询，本地测试运行时切勿关闭终端窗口，不可让电脑进入休眠或断网状态：
打开控制台或终端，进入代码所在目录。
安装核心依赖依赖库：pip3 install py_vsys pandas requests base58
执行启动命令：python vsys_volume_booster.py

# 3. Linux 服务器/VPS 后台静默部署（推荐生产环境使用）
建议将脚本部署于纯净、安全的 Linux VPS 环境隔离目录下静默永续长跑：

1. 进入您的生产工作目录： cd /www/wwwroot/vsys_volume_booster
2. 启动后台静默守护进程（屏蔽挂断信号，输出重定向至 vsys_run.log）： nohup python3 vsys_volume_booster.py > vsys_run.log 2>&1 &
3. 实时追踪链上广播流水日志： tail -f vsys_run.log

# 4.  守护进程安全切断与关闭

1. 查看守护进程的真实 PID 序列号与具体启动时间： ps -ef | grep vsys_volume_booster.py
2. 强行终止该进程（假设查出来的 PID 进程号是 2661514）： kill -9 2661514
3. 再次复检进程是否彻底被物理清除： ps -ef | grep vsys_volume_booster.py

🚀 私钥安全与长效风控隔离红线
绝对的物理隔离，translate_midd.csv（中转站）与 add_priv.csv（独立地址）内包含大量明文地址与私钥。
善后粉碎锁死
当脚本处于停止运行状态时，必须将上述两个包含私钥的 CSV 文件彻底剪切并移动至离线的安全物理 U 盘中，并在联网服务器上使用 Linux shred 命令或安全软件将其粉碎删除。
永续沉淀价值收回凭证
由于本引擎采用长尾随机残留对抗算法，运行一段时间后，每一个散户独立地址（共计多达上万个）内部都会留存 不等的 VSYS 作为对抗风控的沉淀资产。上述包含私钥的外部账本是未来收回这笔沉淀资金的唯一凭证！ 账本一旦丢失，沉淀资金将永远锁死在链上，请务必多重离线冷备份。
